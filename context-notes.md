# 한국 주식 자동매매 시스템 — 컨텍스트 노트

## 프로젝트 개요

코스피 대형주 위주로 외국인 매매 동향을 참고하여 여러 종목을 자동으로 매매하는 시스템.

---

## 확정 항목

| 요소 | 결정값 |
|------|--------|
| 운용 규모 | 1,000만원 |
| 손절 기준 | 진입가 대비 -10% |
| 익절 기준 | 트레일링 스탑 — 고점 대비 -10% 하락 시 청산 |
| API | 한국투자증권 KIS API |
| 언어 | Python |
| 실행 환경 | AWS EC2 t3.micro (서울 리전, 1년 무료) |
| 모니터링 | 텔레그램 봇 (알림 + 수동매매 + 시스템 제어) |
| 선정 종목 수 | 테마당 대장주 2종목, 최대 6종목 |

---

## 핵심 전략

### 매수 3단계 필터

| 단계 | 조건 | 비고 |
|------|------|------|
| 0단계 | 네이버 증권 테마 크롤링 | 당일 등락률 상위 12개 테마 동적 수집 |
| 1단계 | 테마 모멘텀 상위 3개 테마 | 20일 평균 수익률 기준 |
| 2단계 | 현재가 > MA20 > MA60 | 상승추세 확인 |
| 3단계 | 외국인 5일 누적 순매수 > 0 | 외국인 동향 필터 |

- 각 테마에서 위 조건을 모두 만족하는 모멘텀 상위 2종목 선정
- 최대 6종목, 종목당 약 167만원 배분

### 매도 규칙

- 손절: 진입가 대비 -10% 하락 시 즉시 매도
- 트레일링 스탑: 고점 대비 -10% 하락 시 즉시 매도
- 5분마다 보유 종목 모니터링

### 스케줄

- 08:00 — 네이버 테마 크롤링 + 신호 분석 시작
- 09:00 — 장 시작에 맞춰 매수 주문 실행
- 09:00~15:30 — 5분마다 손절/트레일링 스탑 모니터링
- 15:35 — 일일 수익 리포트 텔레그램 전송
- 주말/공휴일 — 자동 스킵

## 테마 수집 방식

### 동적 크롤링 (naver_theme.py)

매일 08:00에 네이버 증권 테마 페이지를 크롤링하여 당일 등락률 상위 12개 테마를 자동 수집. 시장 주도 테마가 바뀌어도 자동으로 반영됨.

**3단계 방어 구조.**

| 단계 | 방법 | 발동 조건 |
|------|------|-----------|
| 1단계 | 네이버 크롤링 (3회 재시도) | 정상 |
| 2단계 | 어제 캐시 (`theme_cache.json`) | 크롤링 실패 시 |
| 3단계 | 하드코딩 기본 10개 테마 | 캐시도 없을 때 |

2, 3단계 발동 시 텔레그램으로 알림 전송.

**종목 필터링 기준.**
- 시총 5,000억 이상
- 종목코드 6자리 (KOSPI 기준)
- 테마당 최대 5종목 수집

### 하드코딩 폴백 테마 (10개)

반도체, 2차전지, 바이오/제약, 방산, 자동차, 금융, 조선/중공업, AI/플랫폼, 철강/소재, 에너지/화학

## 주요 결정 사항

- **외국인 순매수 임계값**: 현재 0원 (순매수면 통과). 모의투자 결과 후 상향 조정 예정.
- **테마당 2종목, 최대 6종목**: 동적 테마 크롤링 도입 후 종목 풀이 넓어져 테마당 2종목으로 확대. 종목당 약 167만원 배분.
- **네이버 테마 크롤링**: KIS API가 테마 분류를 미지원(2021년 이후 관리 중단)하여 네이버 증권 크롤링으로 대체. 3단계 방어로 안정성 확보.
- **KIS 잔고조회 API**: 모의투자 서버 미지원. 실전 전환 시 활성화.
- **외국인 데이터**: KIS 투자자별 매매동향 API 사용 (프로그램 매매 전용 아닌 전체 외국인 매매).
- **토큰 캐시**: KIS 모의투자 토큰 하루 1회 발급 제한으로 파일 캐시 적용.

## 성과 평가 기준

모의투자 결과를 아래 기준으로 판단.

| 지표 | 최소 기준 | 확인 방법 |
|------|-----------|-----------|
| 승률 | 45% 이상 | `/report` |
| 손익비 | 1.5 이상 | `/report` |
| MDD | -20% 이내 | `/report` |
| KOSPI 초과수익 | 플러스 | 수동 비교 |
| 거래 건수 | 30건 이상 | 통계 신뢰도 확보 후 판단 |

평가 일정.

- 1주차: 시스템 안정성 확인 (오류 없이 작동하는지)
- 1개월: 승률/손익비 1차 확인
- 3개월: 전략 유효성 판단 및 파라미터 튜닝
- 6개월 이상: 실전 전환 여부 결정
