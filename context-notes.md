# 한국 주식 자동매매 시스템 — 컨텍스트 노트

## 프로젝트 개요

코스피 대형주 위주로 네이버 증권 테마 크롤링 + 외국인 매매 동향을 참고하여 여러 종목을 자동으로 매매하는 시스템.

---

## 확정 항목

| 요소 | 결정값 |
|------|--------|
| 운용 규모 | 1,000만원 (실현 손익 누적 복리 재투자) |
| 선정 종목 수 | 최대 4종목 (테마 집중 허용) |
| 긴급 손절 | 진입가 대비 -15% → 즉시 매도 (캔들 무시) |
| 손절 | 진입가 대비 -10% + 직전 1분봉 종가 확인 |
| 본전 보호 | +7% 달성 시 스탑을 진입가 +2%로 이동 |
| 모멘텀 약화 익절 | +10% 이상 + 1분봉 하락 전환 시 즉시 매도 |
| 부분 익절 | +20% 달성 시 50% 매도, 잔량 트레일링 |
| 트레일링 스탑 | 고점 대비 -12% + 직전 1분봉 종가 확인 |
| 시간 손절 | 20거래일 경과 + 수익률 +5% 미만 시 강제 청산 |
| API | 한국투자증권 KIS API |
| 언어 | Python |
| 실행 환경 | AWS EC2 t3.micro (서울 리전, 1년 무료) |
| 모니터링 | 텔레그램 봇 (알림 + 수동매매 + 시스템 제어) |

## 핵심 전략

### 매수 4단계 필터

| 단계 | 조건 | 비고 |
|------|------|------|
| 0단계 | KOSPI MA60 시장 추세 필터 | KODEX 200(069500) 종가 > MA60 이면 매수 허용 |
| 1단계 | 네이버 테마 크롤링 | 당일 등락률 상위 12개 테마 수집 → 모멘텀 상위 3개 선정 |
| 2단계 | 현재가 > MA20 > MA60 + 전일 거래량 필터 | 상승추세 + 거래량 확인 |
| 3단계 | 외국인 5일 누적 순매수 > 0 | 외국인 동향 필터 |

- 전체 통과 종목 중 모멘텀 상위 4종목 선정 (테마별 제한 없음)
- 09:15~09:30 눌림목 진입 (1분봉 기반, 09:30 강제 시장가)

### 매도 7단계 로직 (1분봉 기반)

| 단계 | 조건 | 행동 |
|------|------|------|
| 긴급 손절 | 현재가 ≤ 진입가 × 0.85 | 즉시 전량 매도 (캔들 무시) |
| 손절 | 직전 1분봉 종가 ≤ 진입가 × 0.90 | 전량 매도 (whipsaw 방지) |
| 본전 보호 | +7% 달성 시 1회 발동 | 스탑을 진입가 +2%로 이동 |
| 모멘텀 약화 | +10% 이상 + 현재가 < 직전봉 저가 + 현재가 < 5분 MA | 즉시 전량 매도 |
| 부분 익절 | +20% 달성 시 1회 발동 | 보유량 50% 매도, 잔량 트레일링 재개 |
| 트레일링 스탑 | 직전 1분봉 종가 ≤ 고점 × 0.88 | 잔량 전량 매도 |
| 시간 손절 | 20거래일 경과 + 수익 <+5% | 잔량 전량 강제 청산 |

### 스케줄

- 08:00 — 네이버 테마 크롤링 + 신호 분석 시작
- 09:15~09:30 — 1분마다 눌림목 조건 체크 후 매수 (미충족 시 09:30 강제 진입)
- 09:00~15:30 — 2분마다 1분봉 기반 매도 조건 모니터링
- 15:35 — 일일 결산 리포트 + 사후 추적 가격 기록
- 주말/공휴일 — 자동 스킵

## 테마 수집 방식

### 동적 크롤링 (naver_theme.py)

매일 08:00에 네이버 증권 테마 페이지를 크롤링하여 당일 등락률 상위 12개 테마를 자동 수집. 시장 주도 테마가 바뀌어도 자동으로 반영됨.

**3단계 방어 구조.**

| 단계 | 방법 | 발동 조건 |
|------|------|-----------|
| 1단계 | 네이버 크롤링 (3회 재시도) | 정상 |
| 2단계 | 어제 캐시 (`theme_cache.json`) | 크롤링 실패 시 |
| 3단계 | 하드코딩 기본 10개 테마 | 캐시도 없을 때 |

2, 3단계 발동 시 텔레그램으로 알림 전송.

**종목 필터링 기준.**
- 시총 5,000억 이상
- 종목코드 6자리 (KOSPI 기준)
- 테마당 최대 5종목 수집

### 하드코딩 폴백 테마 (10개)

반도체, 2차전지, 바이오/제약, 방산, 자동차, 금융, 조선/중공업, AI/플랫폼, 철강/소재, 에너지/화학

## 에러 모니터링 (error_monitor.py)

| 상황 | 동작 |
|------|------|
| 일반 에러 (현재가 조회 실패 등) | error.log 기록 + 30분에 1회 텔레그램 알림 |
| 심각 에러 (매수/매도 실패, KIS API 오류) | 즉시 텔레그램 알림 (쿨다운 무시) |
| 스케줄러 30분 이상 무응답 | 워치독 스레드가 즉시 텔레그램 알림 |

- RotatingFileHandler: 1MB × 5파일 회전 보관
- `/errors [n]` 명령어로 텔레그램에서 즉시 조회

## 주요 결정 사항

- **1분봉 기반 매수/매도**: 눌림목 진입(매수)과 캔들 종가 확인(매도)으로 고점 추격 매수와 whipsaw 손절을 동시에 방지. 매수는 09:15~09:30 1분 루프, 매도는 2분 루프.
- **긴급 손절 추가**: -15% 이상 급락 시 캔들 확인 없이 즉시 매도. 일반 손절(-10%)은 1분봉 종가 확인 후 매도.
- **모멘텀 약화 조기 익절**: +10% 이상 수익 구간에서 1분봉 하락 전환 시 트레일링보다 먼저 빠져나옴. 고점 부근 탈출 목적.
- **단계별 매도 로직**: 기존 손절+트레일링에서 긴급손절·모멘텀약화·본전보호·부분익절·시간손절 7단계로 확장.
- **트레일링 스탑 완화**: 10% → 12%. 부분 익절 도입으로 이익이 일부 확보되므로 잔량에 더 많은 여유를 줌.
- **복리 재투자**: realized_pnl을 누적해서 다음 매수 기준금 자동 조정. get_available_cash() = 총자산 - 보유 포지션 투자금.
- **종목 선정 방식**: 테마당 2종목 고정 → 전체 통과 종목 중 모멘텀 상위 4종목으로 변경. 특정 테마에 집중될 수 있으나 모멘텀 기반이므로 수익성 우선.
- **외국인 순매수 임계값**: 현재 0원 (순매수면 통과). 모의투자 결과 후 상향 조정 예정.
- **네이버 테마 크롤링**: KIS API가 테마 분류를 미지원(2021년 이후 관리 중단)하여 네이버 증권 크롤링으로 대체. 3단계 방어로 안정성 확보.
- **KIS 잔고조회 API**: 모의투자 서버 미지원. 실전 전환 시 활성화.
- **외국인 데이터**: KIS 투자자별 매매동향 API 사용 (프로그램 매매 전용 아닌 전체 외국인 매매).
- **토큰 캐시**: KIS 모의투자 토큰 하루 1회 발급 제한으로 파일 캐시 적용.
- **에러 모니터링**: 기존 except: pass 구간 전체 교체. 파일 로그 + 심각 에러 텔레그램 즉시 알림으로 운영 중 문제 파악 가능.
- **KIS 모의서버 500 오류 대응**: 매수 주문 5회 재시도(3초 간격). 실패 시 종목을 대기열에 재등록해 1분 후 재시도. 09:30 강제매수도 3회 재시도.
- **매수 시작 시간 09:15**: KIS 모의투자 서버가 09:10~09:14 구간에 집중적으로 500 오류를 반환하는 패턴 확인 후 09:15로 조정.
- **워치독 오경보 수정**: morning_routine이 스케줄러 스레드를 블로킹하는 동안 heartbeat 미갱신 문제 수정. 09:15 대기 후 및 매 루프마다 갱신.
- **CSV 데이터 확장**: 파라미터 튜닝 분석을 위해 trades.csv에 entry_time, exit_time, peak_price, min_price, trigger_price, momentum, foreign_net_buy_mil, ma20/ma60_at_entry, volume_ratio, kospi_trend, dip_entry_used 추가. signal_log.csv에 signal_price, ma20, ma60, volume_ratio 추가.
- **사후 추적 (followup_log.csv)**: 매도 후 3/5/10/20일 가격을 자동 기록. 손절이 너무 빨랐는지, 트레일링이 적절했는지 분석용.
- **베이시스 수집 (basis_log.csv)**: 매일 09:15 KOSPI 200 베이시스(선물-현물)를 네이버에서 스크래핑 후 기록. 매매 조건으로는 미사용, 데이터 축적 후 임계값 결정 예정.
- **타이밍 로그 (timing_log.csv)**: 매수 윈도우(09:15~09:30) 매 분 체크 결과 기록. 눌림목 조건 효과 및 최적 진입 시간 분석용.

## 성과 평가 기준

모의투자 결과를 아래 기준으로 판단.

| 지표 | 최소 기준 | 확인 방법 |
|------|-----------|-----------|
| 승률 | 45% 이상 | `/report` |
| 손익비 | 1.5 이상 | `/report` |
| MDD | -20% 이내 | `/report` |
| KOSPI 초과수익 | 플러스 | 수동 비교 |
| 거래 건수 | 30건 이상 | 통계 신뢰도 확보 후 판단 |

평가 일정.

- 1주차: 시스템 안정성 확인 (오류 없이 작동하는지, error.log 점검)
- 1개월: 승률/손익비 1차 확인
- 3개월: 전략 유효성 판단 및 파라미터 튜닝
- 6개월 이상: 실전 전환 여부 결정
