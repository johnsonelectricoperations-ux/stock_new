### 2026-06-13 (Claude 세션) — Phase B 1~3순위 구현 완료

전략 로직은 변경하지 않음. 시스템 안정성 보강 3건.
1. 유령 포지션 방지 — 매수 주문 예외 시 잔고 조회로 실제 체결 확인, 체결됐으면 신호 메타데이터 보존한 채 포지션 등록 (main.py `_verify_filled_position`). timing_log에 `bought_verified` 액션 추가됨.
2. 시세조회 마비 긴급 알림 — 모니터링 사이클에서 보유 전 종목 조회 실패가 5회(약 10분) 연속되면 텔레그램 긴급 알림, 이후 30분마다 재경보 (6/8~9 마비 재발 대비).
3. CSV 스키마 자동 정합화 — trades.csv·signal_log.csv 기록 시점에 헤더 교체+행 패딩 자동 수행 (performance.py `_ensure_csv_schema`). 실데이터 사본으로 보존·멱등성 검증 완료. 서버에서는 다음 기록 시점에 자동 적용됨.

남은 것. 서버 .env의 STOP_LOSS_RATE(-5% 추정)·EMERGENCY_STOP_RATE 값 확인 후 문서 정합화. 배포 필요 (`./scripts/deploy.sh` — 브랜치 동일하므로 인자 불필요).

### 2026-06-13 (Claude 세션) — Phase A 데이터 진단 분석 완료

성과 (포지션 19건 기준. 레이저쎌 분할매도 합산).
- 승 9 / 패 10 (승률 47%), Profit Factor 2.19, 평균익 +298,828원 / 평균손 -136,543원, 순손익 +1,622,858원
- 실전 전환 기준(승률 45%+, 손익비 1.5+)을 현재까지 충족. 단 표본 부족으로 판정은 30건 후.

핵심 발견.
1. **실제 운영 손절선은 -5%다.** 손절 10건 전부 trigger_price/entry = 0.950. 코드 기본값(-10%)·문서(-10%)와 불일치 → 서버 .env에 STOP_LOSS_RATE=0.05 설정 추정 (사용자 확인 필요). 성과상 -5%는 잘 작동 중 (손절 9건 중 7건이 매도 후 추가 하락, 반등 놓침은 에스에이엠티 +33.5% 1건).
2. **모멘텀약화 룰이 수익의 84%를 생산** (7건, 평균 +13.6%, 합계 +250만원). 이익 반납도 평균 ~2%p로 작음. 매도 후 추적에서도 대부분 추가 하락(LS d20 -34%, 현대모비스 d10 -27%) → 고점 근처 청산 성공. 예외: LG전자 매도 후 d10 +81% (대박 1건 놓침).
3. **사전 가설 기각.** "트레일 -12% 폭이 넓어 이익 반납이 클 것"은 데이터로 기각 — 트레일링 매도는 0건, 모멘텀약화가 항상 먼저 발동. 트레일 폭 축소 검토는 우선순위에서 제외.
4. **부분익절 1건(레이저쎌)은 완벽** — 매도 후 d20 -55% 폭락 전량 회피.

시스템 문제 (심각도 순).
1. **6/8~6/9 시세조회 전면 마비** — KIS 모의서버 포트 문제(9443)로 이틀간 모든 가격 조회 500 에러(366건). 보유 중이던 롯데쇼핑 모니터링이 2일간 작동 불능 → 폭락장이었다면 손절 불가였음. 포트 수정(29443)은 6/9 커밋·금일 배포로 해결됐으나, **API 장애 지속 시 긴급 알림 체계가 없다는 구조적 문제** 확인.
2. **유령 포지션** — timing_log의 buy_failed 21건 중 일부(LG전자 5/15 등)는 실제로는 체결됨. 주문 접수 후 예외 발생 → 실패로 기록·포기 → 잔고에는 존재 → 재시작 복구로 등장. 그 결과 거래 20건 중 8건이 메타데이터(진입시각·섹터·모멘텀) 유실 상태로, 신호-성과 연결 분석이 절반 무력화됨.
3. **trades.csv / signal_log.csv 스키마 드리프트** — 6/9 컬럼 추가 시 기존 파일 헤더 미갱신. 행마다 컬럼 수가 다름(signal_log: 11/15/18열 혼재) → 헤더 마이그레이션 필요.
